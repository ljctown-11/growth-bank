# MEMORY.md - 长期备忘

## 代码推送规则（2026-07-06 起生效）

**禁止自动推送 GitHub / Gitee**
- 未经用户明确指示（如"上传"、"推送"、"push"），绝不执行 `git push` 或任何向远程仓库的上传操作。
- 本地修改和测试阶段，代码仅保留在工作区。
- 用户确认"可以上传了"后再执行推送。
- 同时遵守"执行吧"约定：任何操作先确认，再执行。

## "先分析，再执行"铁律（2026-07-06 起生效）

**所有任务（Bug 修复、新功能、改进建议等）必须遵守以下流程：**

1. 用户提出问题/需求
2. AI 先**分析并给出修改建议**（说明改什么、怎么改、影响范围）
3. 用户确认，明确说"执行吧"（或类似指令）
4. AI 才动手修改文件

**违规说明：**
- 禁止跳过分析直接改文件
- 分析阶段只做方案说明，不涉及文件写入
- 所有文件修改操作必须等待用户明确确认后才执行

## 团队协作铁律（2026-07-06 起生效）

**召唤专家团队时，必须走正式团队流程：**

- 创建团队 → 分派工程师修复 → 分派 QA 验证 → 用户验收
- 严禁主理人自己直接改文件绕过程序
- 所有团队成员的产出必须由该成员亲自完成，不得代写
